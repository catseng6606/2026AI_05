# Antigravity 執行行為約束 (針對教學環境)

## 核心原則

- **禁止自動跨檔案修改**：除非使用者明確指定（例如：修改 index.html 時一併調整 style.css），否則請僅針對當前編輯的檔案進行修改。
- **減少 Agent 多步推理**：優先使用單步代碼生成，不要啟動複雜的 Workspace 掃描任務，以節省 API 配額。

## 技術細節約束

- **Scoped CSS**：當使用者要求修改樣式時，優先建議在當前 HTML 的 `<style>` 標籤中撰寫，或提示使用者手動切換至 .css 檔案。
- **組件化思考**：如果修改內容涉及多個檔案，請在 Chat 視窗中「列出建議修改清單」並說明原因，由使用者手動確認後逐一執行，而非自動直接套用。
- **避免過度搜尋**：不要主動執行 `grep` 或全局搜索整個專案目錄，除非使用者詢問特定變數的定義位置。

## 交互風格

- **解釋優先**：在提供代碼前，先簡短解釋該代碼的作用。
- **引導式修正**：當程式碼出錯時，引導學生思考錯誤原因，而非直接提供完整修正後的檔案。
