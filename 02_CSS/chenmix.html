字體堆疊與 <pre> 標籤應用

透過 <pre> (Preformatted Text) 標籤，我們可以保留程式碼的空白與格式，方便學生對照 CSS 設定。

A. 最佳實踐 (包含 Arial + 微軟正黑體 + 多平台相容)

“Hello 2026” 這是最專業的「中西文混排」效果。 
.font-best {
    /* 1. Arial 處理引號與英數間距 */
    /* 2. 隨後加入 Mac/Win 系統字體提升易讀性 */
    /* 3. 最後由微軟正黑體/萍方處理中文 */
    font-family: Arial, Helvetica, "-apple-system", "Segoe UI", 
                 "Microsoft JhengHei", "PingFang TC", sans-serif;
}

B. 純微軟正黑體 (客戶不滿意的對照組)

“Hello 2026” 這是「純微軟正黑體」的效果。 
.font-msjh {
    /* 缺點：引號會套用中文字體的方塊寬度，導致明顯空白 */
    font-family: "Microsoft JhengHei";
}
💡 教學筆記：為什麼不只寫「Arial + 微軟正黑體」？
雖然這個組合解決了引號問題，但在 Mac 上渲染「微軟正黑體」有時會出現鋸齒或視覺偏移。因此，我們在中間插入了 "-apple-system"。

這並不是移除，而是「進化」：我們確保了 Windows 使用者看到微軟正黑體，Mac 使用者看到萍方，而引號通通交給 Arial。
